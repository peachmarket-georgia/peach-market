## 1. 프로젝트 개요

### 1.1 프로덕트 비전

조지아 한인 커뮤니티의 중고거래를 하나의 플랫폼에 모아, **신뢰할 수 있는 거래 환경**을 만든다.

### 1.2 해결하려는 문제

현재 조지아 한인들은 카카오톡 단체방과 페이스북 그룹에서 중고거래를 하고 있지만, 다음과 같은 근본적인 한계가 있습니다:

- 거래 히스토리 추적 불가
- 거래 상태 관리 불가 (판매중/예약중/완료)
- 게시글 관리 비효율 (메시지에 묻힘)
- 사기 및 분쟁 대응 어려움
- 매물 검색의 어려움

### 1.3 핵심 원칙

**회원제 기반 플랫폼**

- **비회원**: 랜딩페이지에서 더미 매물 리스트로 서비스 가치 체험
- **회원**: 로그인 후 실제 거래 기능(상세보기, 채팅, 등록) 접근 가능

## 4. Phase 1 기능 범위 (Feature Scope)

### 4.1 사용자 관리

#### 요구사항

**1. 이메일/비밀번호 회원가입**

- 필수 입력: 이메일, 비밀번호, 비밀번호 확인, 닉네임, 거주 지역
- 이메일 형식 검증 (정규식)
- 이메일 중복 체크 (실시간)
- 이메일 인증
- 닉네임 중복 체크 (실시간)
- 비밀번호 규칙: 최소 8자, 영문/숫자/특수문자 조합 권장
- 비밀번호 해싱: bcrypt (salt rounds: 10)
- 회원가입 완료 시 JWT 토큰 발급 및 자동 로그인

**2. 이메일/비밀번호 로그인**

- 이메일/비밀번호 입력
- 비밀번호 검증 (bcrypt.compare)
- JWT 세션 관리:
  - **Access Token**: httpOnly cookie에 저장, 유효기간 15분
  - **Refresh Token**: httpOnly cookie에 저장, 유효기간 7일
  - Access Token 만료 시 Refresh Token으로 자동 갱신
  - Refresh Token 만료 시 재로그인 필요

**3. Google OAuth 2.0 (소셜 로그인)**

- [Google로 시작하기] 버튼 제공
- **OAuth 플로우 (NestJS 처리)**:
  1. Next.js에서 NestJS `/auth/google` 엔드포인트 호출
  2. NestJS PassportJS Google Strategy로 OAuth 인증
  3. Google 인증 화면 표시 및 사용자 동의
  4. Google 콜백 → NestJS `/auth/google/callback`
  5. 사용자 정보 조회 (이메일, 이름, 프로필 이미지)
  6. DB에서 이메일로 사용자 검색:
     - 기존 사용자: JWT 발급 및 리다이렉트
     - 신규 사용자: 온보딩 페이지로 리다이렉트 (닉네임/지역 입력)
  7. 온보딩 완료 후 JWT 발급
- Google 이메일은 고유 식별자로 사용
- 비밀번호 필드는 NULL (OAuth 사용자)
- **환경별 Redirect URI 설정 필요**:
  - 로컬: `http://localhost:3003/auth/google/callback`
  - 프로덕션: `https://api.peachmarket.com/auth/google/callback`

**4. 프로필**

- 프로필 필드: 이메일(수정 불가), 닉네임(필수, 2-20자, 중복 불가), 프로필 이미지(선택), 거주 지역(필수)
- OAuth 사용자도 닉네임/지역 설정 필수

**5. 비밀번호 재설정**

- 이메일/비밀번호 사용자만 해당
- 이메일로 재설정 링크 발송
- 토큰 유효시간: 1시간
- 재설정 링크 클릭 시 새 비밀번호 입력 페이지

#### 수락 기준

- [ ] 이메일/비밀번호/닉네임/지역 입력 → 회원가입 완료 → 자동 로그인
- [ ] 이메일 중복 시 에러 메시지 표시
- [ ] 닉네임 중복 시 에러 메시지 표시
- [ ] 비밀번호 규칙 미충족 시 에러 메시지 표시
- [ ] Google 로그인 → OAuth 플로우 → 로그인 완료

### 4.2 게시글 CRUD

#### 요구사항

- **필수 필드**: 제목(최대 50자), 설명(최대 2000자), 가격, 카테고리, 이미지(1-5장), 거래 지역
- **카테고리**: 전자기기, 가구, 의류, 생활용품, 식품, 유아/아동, 자동차, 기타
- **상태**: 판매중(SELLING), 예약중(RESERVED), 판매완료(SOLD)
- **이미지 업로드**:
  - 최대 5장, 각 5MB 이하 (총 25MB 이하)
  - Supabase Storage 저장
  - 허용 포맷: JPG, PNG, WebP
  - 업로드 플로우:
    1. 클라이언트 → NestJS 업로드 엔드포인트
    2. NestJS에서 파일 검증 (크기, 포맷, MIME 타입)
    3. Supabase Storage에 업로드
    4. 썸네일 생성 (Sharp 라이브러리, 300x300px)
    5. URL 배열로 DB 저장
- 가격: USD ($) 표시, DB에는 센트 단위 저장 (정수)
- 무한 스크롤 (cursor pagination, 페이지당 20개)

#### 수락 기준

- [ ] 이미지 포함 게시글 작성 후 목록에 반영

## 5. 보안 및 에러 처리

### 5.1 보안 체크리스트

모든 API 엔드포인트와 기능 구현 시 다음 보안 사항을 필수로 확인합니다.

#### 인증/인가

- [ ] JWT 검증 (NestJS Guard)
- [ ] Refresh Token 로테이션 (재사용 감지)
- [ ] httpOnly 쿠키 사용 (XSS 방지)
- [ ] SameSite=Strict 쿠키 설정 (CSRF 방지)
- [ ] 비밀번호 해싱 (bcrypt, salt rounds: 10)

#### 입력 검증

- [ ] NestJS DTO + class-validator (모든 요청 검증)
- [ ] SQL Injection 방지 (Prisma parameterized queries 자동 처리)
- [ ] XSS 방지 (React 자동 이스케이프 + 사용자 입력 검증)
- [ ] 파일 업로드 검증:
  - MIME 타입 검증 (image/jpeg, image/png, image/webp만 허용)
  - 파일 크기 제한 (5MB)
  - 파일명 sanitization

#### Rate Limiting

- [ ] NestJS Throttler 사용
- [ ] 로그인 시도: 5회/분 (IP 기준)
- [ ] 회원가입: 3회/시간 (IP 기준)
- [ ] API 요청: 100회/분 (사용자 기준)
- [ ] 채팅 메시지: 30회/분 (사용자 기준)

#### CORS

- [ ] NestJS CORS 설정
- [ ] 허용 도메인 화이트리스트:
  - 로컬: `http://localhost:3000`
  - 프로덕션: `https://peachmarket.com`
- [ ] credentials: true (쿠키 전송 허용)

### 5.2 에러 처리 전략

#### 백엔드 (NestJS)

- **Exception Filters**: 통일된 에러 응답 포맷
  ```json
  {
    "statusCode": 400,
    "message": "이메일이 이미 사용 중입니다",
    "error": "Bad Request",
    "timestamp": "2026-02-15T10:30:00.000Z"
  }
  ```
- **HTTP 상태 코드**:
  - 400: 잘못된 요청 (유효성 검증 실패)
  - 401: 인증 실패 (로그인 필요)
  - 403: 권한 없음 (본인 게시글 아님)
  - 404: 리소스 없음
  - 409: 충돌 (이메일/닉네임 중복)
  - 500: 서버 오류

#### 프론트엔드 (Next.js)

- **React Error Boundary**: 렌더링 오류 시 폴백 UI
- **Toast 알림**: 사용자 친화적 에러 메시지
- **네트워크 오류 처리**: 재시도 로직

모든 기능 구현 시 아래 항목을 필수로 확인해야 합니다.

### 6.1 컬러 팔레트 (Pantone Peach Fuzz 기반)

| 역할           | 색상        | Hex       | 용도           |
| -------------- | ----------- | --------- | -------------- |
| Primary        | 피치 오렌지 | `#FF6B35` | CTA 버튼, 강조 |
| Secondary      | 라이트 피치 | `#FFB347` | 보조 요소      |
| Success        | 그린        | `#4CAF50` | 판매중 상태    |
| Warning        | 옐로우      | `#FFC107` | 예약중 상태    |
| Muted          | 그레이      | `#9E9E9E` | 판매완료 상태  |
| Background     | 오프화이트  | `#FAFAFA` | 페이지 배경    |
| Surface        | 화이트      | `#FFFFFF` | 카드 배경      |
| Text Primary   | 블랙        | `#212121` | 제목, 본문     |
| Text Secondary | 그레이      | `#757575` | 보조 텍스트    |

### 6.2 반응형 디자인

모든 페이지와 컴포넌트는 다음 브레이크포인트에서 테스트되어야 합니다:

| 디바이스      | 브레이크포인트 | 우선순위   |
| ------------- | -------------- | ---------- |
| 모바일 (세로) | 320px - 767px  | **최우선** |
| 태블릿        | 768px - 1023px | 중간       |
| 데스크톱      | 1024px 이상    | 하         |

**모바일 최적화 필수 사항**:

- [ ] 터치 영역 최소 44x44px (iOS 가이드라인)
- [ ] 가로 스크롤 없음
- [ ] 폰트 크기: 본문 최소 16px (모바일 줌 방지)
- [ ] 이미지 lazy loading
- [ ] 버튼/입력 필드 충분한 간격 (최소 8px)

### 6.3 성능

- [ ] Lighthouse Performance 점수 90점 이상 (모바일 기준)
- [ ] Core Web Vitals 통과:
  - LCP (Largest Contentful Paint) < 2.5초
  - FID (First Input Delay) < 100ms
  - CLS (Cumulative Layout Shift) < 0.1
- [ ] 이미지 최적화 (WebP, lazy loading)
- [ ] 번들 크기 모니터링 (Vercel Analytics)
