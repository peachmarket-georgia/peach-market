## 프로덕트 비전

조지아 한인 커뮤니티의 중고거래를 하나의 플랫폼에 모아, **신뢰할 수 있는 거래 환경**을 만든다.

## 해결하려는 문제

현재 조지아 한인들은 카카오톡 단체방과 페이스북 그룹에서 중고거래를 하고 있지만, 거래 히스토리 추적 불가, 거래 상태 관리 불가, 게시글 관리 비효율, 사기 및 분쟁 대응 어려움 등의 근본적인 한계가 있다.

## 핵심 원칙

피치마켓은 **회원제 기반** 플랫폼이다. 비회원에게는 랜딩페이지에서 더미 매물 리스트를 보여주어 서비스의 가치를 체험하게 하고, 실제 거래 기능(상세 보기, 채팅, 등록)은 로그인 후 접근 가능하도록 한다.

# 3. 사용자 페르소나

## 페르소나 1: 육아맘 (최대 세그먼트)

- 닉네임에 "맘", "아기", 연령 포함
- 유아용품, 아동복, 도서 거래 중심
- 주 활동 지역: 스와니, 존스크릭, 뷰포드
- 가격대: $10~$50

## 페르소나 2: 이사/귀국 준비자

- "무빙세일", "급히 귀국" 키워드
- 복수 물건 동시 판매, 저가 일괄 판매
- 가구, 생활용품, 전자기기 복합
- 빠른 거래 완료 원함

## 페르소나 3: 신규 입국자

- 닉네임에 "새로온", "초보", "뉴비" 포함
- 주로 구매 요청 ("있나요?", "파시는분?")
- 변압기, 밥솥, 이사박스 등 정착 필수품 찾음

# 6. 사용자 플로우 (User Flow)

## 6.1 비회원 플로우

```
랜딩페이지 접속
    │
    ├─ 히어로 섹션 (CTA: "시작하기")
    │    └─ 클릭 → 로그인 모달
    │
    ├─ 더미 매물 리스트 스크롤
    │    ├─ 카드 클릭 → 로그인 모달
    │    └─ "더 보기" 버튼 → 로그인 모달
    │
    ├─ "물건 등록하기" 버튼
    │    └─ 클릭 → 로그인 모달
    │
    └─ 헤더 "로그인" 버튼
         └─ 클릭 → 로그인 모달
```

## 6.2 회원 플로우

```
로그인 완료
    │
    ├─ [신규] 프로필 온보딩 → 홈 (실제 매물 리스트)
    │
    └─ [기존] 홈 (실제 매물 리스트)
              ├─ 카드 클릭 → 게시글 상세
              ├─ 검색/필터
              ├─ 물건 등록
              ├─ 채팅
              └─ 마이페이지
```

# 7. MVP 기능 범위 (Feature Scope)

## 7.1 사용자 인증 (P0)

### 사용자 스토리

- 사용자는 Google 계정으로 회원가입/로그인할 수 있다
- 사용자는 프로필(닉네임, 이미지, 지역)을 설정할 수 있다
- 사용자는 로그아웃할 수 있다

### 요구사항

- Google OAuth 2.0 기반 소셜 로그인
- 최초 로그인 시 프로필 온보딩 (건너뛰기 불가)
- JWT 세션 관리
- 프로필 수정

### 수락 기준

- [ ] Google 로그인 → OAuth 플로우 → 로그인 완료
- [ ] 최초 가입 시 프로필 설정으로 리다이렉트
- [ ] 비로그인 사용자는 랜딩페이지만 접근 가능

---

## 7.2 게시글 CRUD (P0)

### 사용자 스토리

- 판매자는 게시글을 작성할 수 있다 (제목, 설명, 가격, 카테고리, 이미지)
- 판매자는 게시글을 수정/삭제할 수 있다
- 판매자는 거래 상태를 변경할 수 있다 (판매중/예약중/판매완료/드림)

### 요구사항

- 필드: 제목, 설명, 가격, 카테고리, 이미지(최대 5장), 지역, 상태
- **카테고리 (데이터 기반 정렬):** 가구/인테리어, 유아/아동, 의류/잡화, 도서/교육, 생활용품, 전자기기, 운동/레저, 식품, 자동차, 기타
- 이미지 업로드 (GCP Cloud Storage)
- 무한 스크롤 (cursor pagination)
- 가격: USD ($), **"무료 나눔" 옵션 포함**
- **거래 상태: 판매중 / 예약중 / 판매완료 / 드림(무료)**
- **가격 입력**: 소액 중심이므로 숫자 키패드 우선, 소수점 불필요

### 수락 기준

- [ ] 이미지 포함 게시글 작성 후 목록에 반영
- [ ] 거래 상태 변경 시 실시간 반영
- [ ] 본인 게시글만 수정/삭제 가능
- [ ] 이미지 최대 5장, 각 5MB 이하
- [ ] "드림" 상태 설정 시 가격 $0으로 자동 전환

## 7.3 검색 및 필터 (P0)

### 사용자 스토리

- 키워드로 게시글 검색
- 카테고리/상태별 필터링
- 최신순/가격순 정렬

### 요구사항

- 한국어 키워드 검색

- 정렬: 최신순(기본), 가격 낮은순, 가격 높은순

## 7.4 1:1 채팅 (P0 — 데이터 기반 승격)

- 텍스트 메시지 전송
- 채팅 목록 (최근 대화순)
- 읽음/안읽음 표시
- 새 메시지 알림 (인앱)

### 수락 기준

- [ ] 게시글에서 "채팅하기" → 채팅방 생성
- [ ] 메시지 전송 및 실시간 수신
- [ ] 읽지 않은 메시지 뱃지 표시

# 8. 페이지별 상세 설계

## 8.1 랜딩페이지 (`/`)

**목적:** 비회원에게 서비스 가치를 보여주고 회원가입을 유도한다.

### 레이아웃 (위에서 아래로)

**① 헤더 (고정)**

- 왼쪽: 피치마켓 로고 + "피치마켓"
- 오른쪽: **"로그인"** 버튼 (Primary)
- 모바일: 햄버거 메뉴 아이콘 → 로그인 버튼

**② 히어로 섹션**

- 헤드라인: "조지아 한인 커뮤니티의 신뢰할 수 있는 중고거래"
- 서브텍스트: "카카오톡방, 페이스북에 흩어진 매물을 한곳에서"
- CTA 버튼: **"시작하기"** → 로그인 모달
- 배경: 피치 컬러 그라디언트 또는 조지아 일러스트

**③ 더미 매물 리스트 섹션**

- 섹션 타이틀: "지금 올라온 매물"
- 더미 데이터 카드 12개 표시 (2열 그리드)
- 카드 클릭 시: 로그인 모달 표시 ("매물 상세를 보려면 로그인이 필요합니다")

**② 정보 섹션** — 상태 뱃지 + 카테고리 태그, 제목, 가격 (Primary Bold), 지역 · 시간, 설명

**③ 판매자 정보 카드** — 프로필 + 닉네임 + 지역 + 거래 완료 수 뱃지

**④ 하단 고정 액션 바**

- 오른쪽: **"채팅하기"** 버튼

## 8.6 게시글 등록 (`/posts/new`)

**진입 조건:** 로그인 필수

### 폼 필드 (위에서 아래로)

1. **이미지 업로드 영역** — 최대 5장, 드래그 정렬, 첫번째 = 대표
2. **제목** (required, 최대 50자)
3. **카테고리** (required, 데이터 기반 정렬 드롭다운)
4. **가격** (required, 숫자 키패드, $ 자동 표시, **"무료 나눔" 체크박스**)
5. **거래 희망 지역** (required, 데이터 기반 정렬 드롭다운)
6. **설명** (required, 최대 2000자)
7. **"등록하기"** 버튼 — 필수 필드 미입력 시 비활성화

# 11. 기술 구현 노트

## 11.1 라우팅 구조 (Next.js App Router)

```
app/
├─ (public)/          ← 비회원 접근 가능
│  ├─ page.tsx        ← 랜딩페이지
│  └─ layout.tsx
├─ (auth)/            ← 인증 관련
│  ├─ signin/
│  └─ onboarding/
├─ (protected)/       ← 로그인 필수 (middleware)
│  ├─ home/           ← 매물 리스트
│  ├─ posts/
│  │  ├─ new/
│  │  └─ [id]/
│  │     └─ edit/
│  ├─ search/
│  ├─ chat/
│  │  └─ [roomId]/
│  ├─ profile/
│  │  ├─ posts/
│  │  ├─ bookmarks/
│  │  └─ edit/
│  └─ users/
│     └─ [id]/
└─ api/
```

## 11.2 Middleware 로직

```
middleware.ts
│
├─ public 경로 ("/")       → 로그인 상태면 /home 리다이렉트
├─ auth 경로              → 로그인 상태면 /home 리다이렉트
├─ onboarding 경로       → 온보딩 완료면 /home 리다이렉트
└─ protected 경로        → 비로그인이면 / 리다이렉트
                          → 온보딩 미완료면 /auth/onboarding 리다이렉트
```

## 11.3 더미 데이터 전략

- 랜딩페이지의 더미 매물은 **정적 JSON 파일**로 관리
- 파일 위치: `src/data/dummy-posts.json`
- 실제 데이터 기반 12개 아이템 (섹션 8.1 참고)
- 추후 실제 데이터로 전환 시 쉽게 교체 가능하도록 추상화

## 11.4 시간대 기반 UX 전략

- 새 게시글 정렬: 단순 시간순이 아닌 **활동 시간대 가중치** 고려
- 추후 푸시 알림: **오후 9시~자정 EST** 집중 발송
- 채팅 응답 시간 기대치: 새벽에는 느릴 수 있음을 UI로 안내

# 16. 전체 수락 기준 (Acceptance Criteria)

## 랜딩페이지

- [ ] 비회원이 더미 매물 12개 확인 가능
- [ ] 더미 카드 클릭 시 로그인 모달 표시
- [ ] 히어로 CTA 클릭 시 로그인 모달 표시
- [ ] 로그인 상태로 접속 시 /home으로 자동 이동

- [ ] 온보딩 미완료 시 protected 경로 차단
- [ ] 모바일 Bottom Sheet 로그인 모달

- [ ] 이미지 포함 게시글 작성 후 목록 반영

- [ ] 검색 + 필터 동시 적용
- [ ] 가격 필터 레인지 정상 동작
- [ ] 드림 전용 필터 정상 동작
